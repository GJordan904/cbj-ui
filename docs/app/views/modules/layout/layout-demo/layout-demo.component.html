Layout Component Providing General Structure and App Wide Navigation

Overview

Every app needs an app bar and navigation, this component provides both elements as well as a responsive base structure for your app. It provides a navigation bar at the top, a slide out sidebar, and a scrollable main content area. This demo site is utilizing the Layout Component.

Navbar

The navbar is a standard bootstrap navbar with the exception that it has a fixed height. This can be set with a SCSS variable, aptly named, $navbar-height. Additional classes may be added to the navbar with the navbarClasses attribute.

The navbar has two slots for content, navbar-brand and navbar-content.

Sidebar

The sidebar basically has two forms, mobile and desktop. In mobile view, the sidebar is absolutely positioned and slides in from the left on toggle. While in desktop view, we leverage the power of Bootstrap and flexbox and simply expand and collapse the sidebar.

The sidebar also has two slots for content, sidebar-header, and sidebar-footer. Additionally, the sidebar can be disabled all together with the useSidebar attribute.

Slots

I use slots to provide a way to add content into specific areas of the layout, there are a total of five slots. Two slots are in the navbar, two are in the sidebar, and the last is the main content area. Below is the Layout Components template to help visualize.

Usage

The usage is pretty straightforward. Add the component to a root view like app.component, add your content inside some or all of the 5 available slots, then configure the sidebar if you are using it.

Sidebar Configuration

Should you choose to use the sidebar, you will need to pass a SidebarLinks Object to the sidebarLinks attribute. This object will define the links you would like in the sidebar. There are 3 types of links, they are defined as SidebarLinkTypes enum and described in the API section below.
