Once installed, CBJ UI needs to be imported into your app module using the forRoot() method:

This will register the providers and make all the modules available within the app module. Should you need the functionality in other modules, you can import individual modules as needed or simply import the CbjUiModule again but be sure to only use the forRoot() method in the App Module.

Shared Module

If you have multiple modules across which you want to use this package, a recommended pattern would be to implement a shared module. First, import the CbjUiModule into your App Module as outlined above. Then, create a shared module for importing into all your sub-modules. Be sure to export any modules you wish to share from this module.

In the example above I export the FormsModule and RouterModule simply for example, it is not necessary to export these modules but they are pretty common and usually find their way into my shared module.

Bootstrap

CBJ UI was built to be used with Bootstrap 4 during the development of your project. The styles are in SCSS and make extensive use of the Bootstrap mixins and variables. This makes for easy theme customization with the right configuration.

The intended way of using the styles would be to import them into your main SCSS file right after importing Bootstrap. This way you can import your own variables file before Bootstrap and override any Bootstrap or CBJ UI settings.
